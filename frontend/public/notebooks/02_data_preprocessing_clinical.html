Overview

This notebook contains the preprocessing pipeline for the clinical metadata from the GSE96058 GEO dataset. The main steps include:

  1. Data Download & Extraction: The dataset is downloaded and relevant metadata is extracted.
  2. Cleaning: Constant columns are dropped, and missing values (represented as "NA") are standardized to np.nan.
  3. Feature Exploration: Columns are inspected to understand their types and contents.
  4. Preprocessing:
    • Numeric features are converted to numeric types.
    • Binary features are mapped to 0/1.
    • Ordinal categorical features are label encoded to preserve order.
    • Nominal categorical features are one-hot encoded.
  5. Column Management: Unnecessary columns and duplicates are dropped, and columns are renamed for consistency.
  6. Export: Cleaned and one-hot encoded datasets are exported for downstream analysis.
  7. Imputation: Missing values in selected columns are imputed using KNN, and the imputed dataset is exported.

This workflow ensures the clinical data is ready for exploratory data analysis (EDA) and machine learning applications.

Data Preprocessing

Preprocessing of the features that remain:

  • Numeric Features: leave as is, possibly standardize later
  • Binary Features: map to 0/1
  • Ordinal Categorical (e.g. nhg): map to 1,2,3
  • Nominal categorical (e.g. pam50 subtype): one-hot encode -> so we don't introduce an artificial order

def find_repo_root(start_path: Path = None) -> Path:
    if start_path is None:
        start_path = Path().resolve()
    for parent in [start_path] + list(start_path.parents):
